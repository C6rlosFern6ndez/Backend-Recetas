Necesito practicar mis conocimientos en java y sql, vamos a desarrollar una api 
rest para un catalogo de recetas, primero vamos a preparar la estructura de la bbdd 
para mariadb y una vez este completa realizaremos un crud en java para probarla 
incluyendo swagger para las pruebas.
Esto es lo que llevo por el momento de la bbdd:
- Tablas.sql
- Usuarios.sql
- categorias.sql
- ingredientes.sql
- recetas.sql
- pasos.sql
- receta_categorias.sql
- receta_ingredientes.sql
- comentarios.sql
- calificaciones.sql

Dame las demas tablas siguiendo el mismo formato, piensa con cuidado en las relaciones
para mantener la consistencia.

Estructura del proyecto MVC:
src/
├── main/
│   ├── java/
│   │   ├── com/
│   │   │   ├── recetas/
│   │   │   │   ├── model/        // Clases de modelos (entidades)
│   │   │   │   ├── controller/   // Controladores para manejar solicitudes HTTP
│   │   │   │   ├── repository/   // Repositorios para interactuar con la base de datos
│   │   │   │   ├── service/      // Servicios con lógica de negocio
│   │   │   │   └── config/       // Configuración de la aplicación
│   │   │   └── recetas/
│   │   │       └── utils/        // Clases utilitarias (opcional)
│   └── resources/
│       ├── static/               // Recursos estáticos (CSS, JavaScript, imágenes)
│       └── templates/            // Plantillas (si se usan)
└── test/
    └── java/
        └── com/
            └── recetas/
                └── ...            // Pruebas unitarias y de integración
